已在项目里把 MVP 的 Flutter 应用层实现出来了，核心包括本地 SQLite 存储、Riverpod 状态管理、拍照/相册录入、LLM 解析与追问、确认卡片、搜索、浏览、详情编辑和删除。主入口和主题在 [lib/main.dart](/data/projects/wherehouse/lib/main.dart) 与 [lib/app.dart](/data/projects/wherehouse/lib/app.dart)，数据层在 [lib/database/database.dart](/data/projects/wherehouse/lib/database/database.dart) 和 [lib/database/daos/thing_dao.dart](/data/projects/wherehouse/lib/database/daos/thing_dao.dart)，LLM 与图片存储在 [lib/services/llm_service.dart](/data/projects/wherehouse/lib/services/llm_service.dart) 和 [lib/services/storage_service.dart](/data/projects/wherehouse/lib/services/storage_service.dart)。录入、首页、浏览、详情页分别在 [lib/screens/add_screen.dart](/data/projects/wherehouse/lib/screens/add_screen.dart)、[lib/screens/home_screen.dart](/data/projects/wherehouse/lib/screens/home_screen.dart)、[lib/screens/browse_screen.dart](/data/projects/wherehouse/lib/screens/browse_screen.dart)、[lib/screens/detail_screen.dart](/data/projects/wherehouse/lib/screens/detail_screen.dart)。环境变量模板也按 ADR 改成了 [\.env.example](/data/projects/wherehouse/.env.example)。

实现上做了两个关键处理。第一，`drift` 走原生 SQL 封装，不依赖代码生成，避免当前环境缺少 Dart/Flutter 时卡死在 build runner。第二，Dashscope 调用失败或没配 `DASHSCOPE_API_KEY` 时，会自动回退到本地规则提取，所以录入链路依然可用。

当前限制需要明确说明：这台机器没有 `flutter` / `dart` 命令，所以我没法运行 `flutter analyze`、`flutter test`，也没法用 `flutter create` 生成标准的 `android/` 和 `ios/` 原生骨架。代码和测试文件已经补齐，但原生平台目录与实际编译验证需要在有 Flutter SDK 的环境里完成。
