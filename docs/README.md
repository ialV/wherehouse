# Project Documentation

This directory holds project notes that are useful for development but do not need to live in the repository root.

## Layout

- `adr/`: architecture decision records and product/technical decisions.
- `handoff.md`: current project state, verification status, and next likely tasks.
- `history/`: historical prompts, dispatch summaries, and implementation notes from earlier coding rounds.
- `product-direction.md`: reusable product conclusions and positioning notes.

Keep root-level files limited to active project configuration, contributor guidance, and files required by Flutter or CI.
